1. Introduction
   1. Purpose
เอกสาร Software Requirements Specification (SRS) ฉบับนี้จัดทำขึ้นเพื่ออธิบายความต้องการของระบบ LHKEM (Loeng Him Kow Enrollment & Map) ซึ่งเป็น Platform เว็บสำหรับรวบรวมกิจกรรม Workshop และข้อมูลการท่องเที่ยวเชิงชุมชนจากหลายชุมชนไว้ในระบบกลางเดียวกัน โดยเอกสารนี้มีวัตถุประสงค์เพื่อกำหนดขอบเขต ฟังก์ชันการทำงาน และข้อกำหนดของระบบอย่างชัดเจน เพื่อใช้เป็นแนวทางร่วมกันในการพัฒนา ออกแบบ ทดสอบ และประเมินระบบให้ตรงตามความต้องการของผู้มีส่วนเกี่ยวข้องทุกฝ่าย


เอกสารฉบับนี้จัดทำขึ้นสำหรับกลุ่มผู้อ่านหลัก ได้แก่ ทีมพัฒนาระบบ ผู้สอนและผู้ประเมินรายวิชา Software Engineering ผู้ดูแลระบบ Platform (Platform Admin) ผู้ดูแลชุมชน (Community Admin) รวมถึงผู้ที่เกี่ยวข้องกับการนำระบบไปใช้งานจริงในอนาคต เพื่อให้ทุกฝ่ายมีความเข้าใจตรงกันเกี่ยวกับโครงสร้าง บทบาทผู้ใช้งาน และพฤติกรรมของระบบ
   2. Scope
ระบบ LHKEM ถูกออกแบบให้รองรับผู้ใช้งานหลายกลุ่ม โดยแต่ละกลุ่มมีบทบาท หน้าที่ และสิทธิ์การใช้งานที่แตกต่างกัน เพื่อให้การจัดการระบบเป็นไปอย่างเป็นระเบียบและเหมาะสมกับบริบทของPlatformที่รองรับหลายชุมชน ผู้ใช้งานหลักของระบบสามารถแบ่งออกเป็นกลุ่มดังต่อไปนี้
1. นักท่องเที่ยว (Tourist / General User)
นักท่องเที่ยวเป็นผู้ใช้งานทั่วไปที่เข้ามาใช้งานระบบเพื่อค้นหาข้อมูลและเข้าร่วมกิจกรรมในชุมชนต่าง ๆ บนPlatform ความสามารถหลักของนักท่องเที่ยว ได้แก่ 
   1. สมัครสมาชิกและเข้าสู่ระบบเพื่อใช้งาน Platform
   2. เลือกดูข้อมูลชุมชน ร้านค้า และกิจกรรมจากหลายชุมชนภายในระบบเดียว
   3. ค้นหาและดูรายละเอียด Workshop และ Event
   4. ลงทะเบียนเข้าร่วม Workshop และตรวจสอบสถานะการลงทะเบียน
   5. ดูแผนที่ชุมชนและตำแหน่งของร้านค้าและสถานที่จัดกิจกรรม
   6. รับการแจ้งเตือนผ่านอีเมลเกี่ยวกับการลงทะเบียนหรือการเปลี่ยนแปลงกิจกรรม
2. ผู้ดูแลระบบPlatform (Platform Admin)
ผู้ดูแลระบบPlatformเป็นผู้ใช้งานระดับสูงสุด มีหน้าที่ดูแลภาพรวมของPlatformทั้งหมด และควบคุมโครงสร้างหลักของระบบ ความสามารถหลักของ Platform Admin ได้แก่
   7. เพิ่ม แก้ไข และจัดการชุมชนภายในPlatform
   8. กำหนดและจัดการสิทธิ์ของผู้ดูแลชุมชน (Community Admin)
   9. ตรวจสอบภาพรวมของกิจกรรมและการใช้งานระบบในระดับPlatform
   10. ดูแลความถูกต้องและความเรียบร้อยของข้อมูลในระบบโดยรวม
3. ผู้ดูแลชุมชน (Community Admin)
ผู้ดูแลชุมชนเป็นผู้ใช้งานที่รับผิดชอบการดูแลชุมชนของตนเองภายในPlatform โดยไม่สามารถเข้าถึงหรือจัดการข้อมูลของชุมชนอื่นได้ ความสามารถหลักของ Community Admin ได้แก่
   11. จัดการข้อมูลหน้าโปรไฟล์ของชุมชน เช่น ประวัติ ข้อมูลติดต่อ และจุดเด่น
   12. อนุมัติ แก้ไข หรือจัดการ Workshop และ Event ที่สร้างโดยร้านค้าในชุมชน
   13. จัดการร้านค้าและกิจกรรมภายในชุมชนของตนเอง
4. ร้านค้า / ผู้จัดกิจกรรม (Shop / Vendor)
ร้านค้าหรือผู้จัดกิจกรรมเป็นผู้ใช้งานที่มีหน้าที่สร้างและบริหาร Workshop หรือกิจกรรมภายในชุมชนที่ตนเองสังกัด ความสามารถหลักของร้านค้า ได้แก่
   14. สร้างและจัดการโปรไฟล์ร้านค้า
   15. สร้าง แก้ไข หรือยกเลิก Workshop ของตนเอง
   16. กำหนดรายละเอียดกิจกรรม เช่น วันเวลา จำนวนที่นั่ง และสถานะการเปิดรับสมัคร
   17. ตรวจสอบรายชื่อและข้อมูลการลงทะเบียนของนักท่องเที่ยว
   18. สื่อสารข้อมูลที่เกี่ยวข้องกับกิจกรรมผ่านระบบที่กำหนดไว้

   1. Overview
เอกสาร Software Requirements Specification (SRS) ฉบับนี้อธิบายความต้องการของระบบ LHKEM (Loeng Him Kaw Enrollment & Map) ซึ่งเป็นPlatformเว็บสำหรับรวบรวมข้อมูลกิจกรรม Workshop ร้านค้า และข้อมูลการท่องเที่ยวเชิงชุมชนจากหลายชุมชนไว้ในระบบกลางเดียวกัน โดยเอกสารจัดทำตามโครงสร้างของมาตรฐาน IEEE 830-1998 และปรับให้เหมาะสมกับบริบทของโครงการ


เอกสารประกอบด้วยส่วน Introduction ซึ่งอธิบายวัตถุประสงค์ ขอบเขต คำจำกัดความ และเอกสารอ้างอิงของระบบ ตามด้วยส่วน Overall Description ที่อธิบายภาพรวมของระบบ กลุ่มผู้ใช้งาน และข้อจำกัดต่าง ๆ ของระบบ ส่วน Specific Requirements ระบุข้อกำหนดของระบบทั้งในเชิงฟังก์ชันและไม่เชิงฟังก์ชัน และส่วน Supporting Information ซึ่งรวบรวมข้อมูลสนับสนุน เช่น ตารางสรุปฟีเจอร์ แผนโครงการ และภาคผนวก

2. Overall description
   1. Product perspective
      1. System interfaces
ระบบ LHKEM มีการเชื่อมต่อกับระบบภายนอกเพื่อสนับสนุนการทำงานของPlatform ดังนี้
* ระบบแผนที่ภายนอก เพื่อแสดงตำแหน่งของชุมชน ร้านค้า และสถานที่จัดกิจกรรม
* ระบบแจ้งเตือน เพื่อส่งข้อมูลสถานะต่าง ๆ ให้กับผู้ใช้งาน เช่น การลงทะเบียนกิจกรรมหรือการอัปเดตข้อมูล
* การเชื่อมต่อทั้งหมดดำเนินการผ่าน API โดยระบบ LHKEM ทำหน้าที่เป็นผู้เรียกใช้งาน (API consumer) และจัดเก็บข้อมูลที่จำเป็นไว้ภายในฐานข้อมูลของระบบ
      1. User interfaces
ระบบ LHKEM มีส่วนติดต่อผู้ใช้ในรูปแบบเว็บแอปพลิเคชัน (Web-based User Interface) ที่ออกแบบให้ใช้งานผ่านเว็บเบราว์เซอร์ทั่วไป โดยมุ่งเน้นความเรียบง่ายและความสะดวกในการใช้งานสำหรับผู้ใช้ทุกกลุ่ม ระบบมีการแยกส่วนติดต่อผู้ใช้ตามบทบาทของผู้ใช้งาน เช่น นักท่องเที่ยว ร้านค้า ผู้ดูแลชุมชน และผู้ดูแลPlatform เพื่อให้แต่ละกลุ่มสามารถเข้าถึงฟังก์ชันที่เกี่ยวข้องกับหน้าที่ของตนเองได้อย่างเหมาะสม


ส่วนติดต่อผู้ใช้ถูกออกแบบให้รองรับการใช้งานบนอุปกรณ์หลากหลายประเภท เช่น คอมพิวเตอร์ แท็บเล็ต และโทรศัพท์มือถือ (Responsive Design) โดยเน้นการนำเสนอข้อมูลที่ชัดเจน เข้าใจง่าย และสอดคล้องกับลักษณะการใช้งานของระบบท่องเที่ยวเชิงชุมชน
________________
      2. Software interfaces
ระบบมีการเชื่อมต่อกับบริการแผนที่ภายนอก เช่น Google Maps API เพื่อใช้แสดงตำแหน่งของชุมชนและพิกัดทางภูมิศาสตร์ในระดับภาพรวม นอกจากนี้ ระบบมีการเชื่อมต่อกับฐานข้อมูลเพื่อจัดเก็บและจัดการข้อมูลผู้ใช้งาน ชุมชน ร้านค้า Workshop และกิจกรรมต่าง ๆ ภายในPlatform


การสื่อสารระหว่างส่วนต่าง ๆ ของระบบดำเนินการผ่าน API ภายในระบบ เพื่อให้สามารถแยกส่วนการทำงานระหว่างส่วนติดต่อผู้ใช้ ส่วนประมวลผล และส่วนจัดเก็บข้อมูลได้อย่างเป็นระบบ และรองรับการขยายหรือปรับปรุงระบบในอนาคต
      3. Memory constraints
ข้อมูลที่จัดเก็บในระบบประกอบด้วยข้อมูลผู้ใช้งาน ข้อมูลชุมชน ร้านค้า Workshop กิจกรรม และข้อมูลที่เกี่ยวข้องกับการลงทะเบียน ซึ่งจะถูกจัดเก็บในฐานข้อมูลของระบบอย่างเป็นโครงสร้าง นอกจากนี้ ระบบอาจมีการจัดเก็บไฟล์ประกอบ เช่น รูปภาพของชุมชน ร้านค้า หรือกิจกรรม โดยมีการจำกัดขนาดและประเภทของไฟล์ที่สามารถอัปโหลดได้ เพื่อควบคุมการใช้พื้นที่จัดเก็บและประสิทธิภาพของระบบ


ระบบถูกออกแบบให้สามารถทำงานได้บนสภาพแวดล้อมเซิร์ฟเวอร์มาตรฐาน โดยไม่ต้องใช้หน่วยความจำหลัก (RAM) ในปริมาณสูงเกินความจำเป็น และสามารถปรับขยายทรัพยากรได้ตามปริมาณการใช้งานในอนาคต
________________
      4. Operations
นักท่องเที่ยวสามารถเข้าถึงฟีเจอร์พื้นฐาน เช่น การดูข้อมูลชุมชน ร้านค้า กิจกรรม Workshop การลงทะเบียนเข้าร่วมกิจกรรม และการดูแผนที่ชุมชน ร้านค้าและผู้จัดกิจกรรมสามารถเข้าถึงฟีเจอร์สำหรับการสร้างและจัดการWorkshopหรือกิจกรรมของตนเอง ผู้ดูแลชุมชนสามารถเข้าถึงฟีเจอร์สำหรับการจัดการข้อมูลชุมชน ร้านค้า และกิจกรรมภายในชุมชนของตน ส่วนผู้ดูแลPlatformสามารถเข้าถึงฟีเจอร์สำหรับการจัดการภาพรวมของระบบและชุมชนทั้งหมด


ระบบถูกออกแบบให้รองรับการใช้งานตลอดเวลา โดยการดำเนินงานของระบบขึ้นอยู่กับการเชื่อมต่อเครือข่ายอินเทอร์เน็ตและความพร้อมใช้งานของบริการที่เกี่ยวข้อง
      5. Site adaptation requirements
ระบบไม่จำเป็นต้องติดตั้งซอฟต์แวร์เพิ่มเติมบนอุปกรณ์ของผู้ใช้งาน และสามารถปรับการแสดงผลให้เหมาะสมกับขนาดหน้าจอและความสามารถของอุปกรณ์แต่ละประเภท เพื่อรองรับการใช้งานในบริบทที่แตกต่างกัน เช่น การใช้งานในพื้นที่ชุมชนหรือการใช้งานจากสถานที่ภายนอก


การปรับใช้ระบบสามารถดำเนินการได้บนโครงสร้างพื้นฐานเซิร์ฟเวอร์มาตรฐาน และไม่ขึ้นกับสถานที่ตั้งทางกายภาพของผู้ใช้งาน ทำให้ระบบสามารถรองรับการใช้งานจากหลายพื้นที่และหลายชุมชนภายในPlatformเดียวกัน

   2. Product functions
ระบบ LHKEM (Loeng Him Kaw Enrollment & Map) เป็นเว็บแพลตฟอร์มที่ออกแบบมาเพื่อรวบรวมข้อมูลชุมชน เวิร์กช็อป และกิจกรรมเชิงท่องเที่ยวจากหลายชุมชนไว้ในระบบกลางเดียวกัน โดยมีหน้าที่หลักดังต่อไปนี้
* ให้ผู้ใช้งานทั่วไปสามารถค้นหา ดูข้อมูล และลงทะเบียนเข้าร่วมเวิร์กช็อปหรือกิจกรรมจากหลายชุมชนภายในแพลตฟอร์มเดียว
* แสดงข้อมูลชุมชน ร้านค้า และกิจกรรมต่าง ๆ พร้อมแผนที่ชุมชนแบบโต้ตอบ เพื่อช่วยในการตัดสินใจและการเดินทาง
* รองรับระบบบัญชีผู้ใช้เดียวที่สามารถใช้งานได้กับทุกชุมชนในแพลตฟอร์ม
* เปิดโอกาสให้ร้านค้าหรือผู้จัดกิจกรรมสามารถสร้างและจัดการเวิร์กช็อปของตนเองได้
* สนับสนุนการบริหารจัดการชุมชนผ่านระบบผู้ดูแลชุมชน (Community Admin) เช่น การจัดการร้านค้าและกิจกรรมภายในชุมชน
* สนับสนุนการบริหารจัดการแพลตฟอร์มในภาพรวมผ่านผู้ดูแลแพลตฟอร์ม (Platform Admin) เช่น การเพิ่มชุมชนและกำหนดสิทธิ์ผู้ใช้งาน
* แจ้งเตือนผู้ใช้งานเกี่ยวกับสถานะการลงทะเบียนหรือการเปลี่ยนแปลงกิจกรรมผ่านระบบอีเมล
* ระบบถูกออกแบบมาเพื่อสนับสนุนการท่องเที่ยวเชิงชุมชน เพิ่มความสะดวกในการเข้าถึงข้อมูล และช่วยให้การจัดการกิจกรรมภายในชุมชนเป็นไปอย่างมีประสิทธิภาพ

   3. User characteristics
ผู้ใช้งานของระบบ LHKEM สามารถแบ่งออกเป็นหลายกลุ่มตามบทบาทและระดับการเข้าถึงระบบ โดยแต่ละกลุ่มมีลักษณะการใช้งานและความต้องการที่แตกต่างกัน ดังนี้
* ผู้เยี่ยมชม / นักท่องเที่ยว (Visitor / General User)
เป็นผู้ใช้งานทั่วไปที่เข้ามาใช้งานระบบเพื่อค้นหาข้อมูลชุมชน ร้านค้า และกิจกรรมต่าง ๆ โดยอาจเป็นทั้งผู้ที่สมัครสมาชิกและผู้ที่ยังไม่ได้สมัครสมาชิก ผู้ใช้งานกลุ่มนี้คาดหวังการใช้งานที่ง่าย ไม่ซับซ้อน และสามารถเข้าถึงข้อมูลที่จำเป็นได้อย่างรวดเร็ว


* ผู้ดูแลPlatform (Platform Admin)
เป็นผู้ใช้งานระดับสูงสุด มีความรู้พื้นฐานด้านการใช้งานระบบเว็บและการจัดการข้อมูล ทำหน้าที่ดูแลภาพรวมของPlatform เช่น การเพิ่มและจัดการชุมชน การกำหนดสิทธิ์ของผู้ดูแลชุมชน และการตรวจสอบความถูกต้องของข้อมูลในระดับPlatform


* ผู้ดูแลชุมชน (Community Admin)
เป็นผู้แทนหรือกรรมการของแต่ละชุมชน มีความคุ้นเคยกับข้อมูลภายในชุมชนของตนเอง ทำหน้าที่ดูแลและจัดการข้อมูลชุมชน ร้านค้า และกิจกรรมภายในพื้นที่ที่รับผิดชอบ โดยไม่สามารถเข้าถึงข้อมูลของชุมชนอื่นได้


* ร้านค้า / ผู้จัดกิจกรรม (Shop / Vendor)
เป็นผู้ใช้งานที่สร้างและบริหารจัดการWorkshopหรือกิจกรรมภายในชุมชนที่ตนสังกัด มีทักษะพื้นฐานในการใช้งานเว็บแอปพลิเคชัน และใช้งานระบบเพื่อจัดการข้อมูลกิจกรรม การรับสมัคร และการตรวจสอบข้อมูลผู้ลงทะเบียน
________________
   4. Constraints
ระบบ LHKEM มีข้อจำกัดด้านกระบวนการชำระเงิน โดยระบบไม่รองรับการตรวจสอบหรือยืนยันสลิปการโอนเงินโดยอัตโนมัติ การยืนยันการชำระเงินสำหรับการจองกิจกรรมหรือWorkshopจึงขึ้นอยู่กับกระบวนการที่กำหนดไว้ เช่น การตรวจสอบโดยผู้จัดกิจกรรมหรือผู้ดูแลชุมชน


ข้อจำกัดดังกล่าวส่งผลให้ระบบมุ่งเน้นการจัดการข้อมูลการลงทะเบียนและการสื่อสารสถานะกิจกรรม มากกว่าการประมวลผลธุรกรรมทางการเงินโดยตรง
   5. Assumptions and dependencies
การดำเนินงานของระบบ LHKEM ตั้งอยู่บนสมมติฐานว่าผู้ดูแลชุมชน ร้านค้า และผู้จัดกิจกรรมภายในแต่ละชุมชนให้ความร่วมมือในการจัดเตรียม ดูแล และปรับปรุงข้อมูลที่เกี่ยวข้องกับชุมชน ร้านค้า และกิจกรรมให้มีความถูกต้องและเป็นปัจจุบัน


นอกจากนี้ ระบบมีความจำเป็นต้องพึ่งพาความพร้อมใช้งานของโครงสร้างพื้นฐานด้านเครือข่ายอินเทอร์เน็ต และบริการภายนอกที่เกี่ยวข้อง เช่น ระบบอีเมลและบริการแผนที่ เพื่อให้ระบบสามารถให้บริการแก่ผู้ใช้งานได้อย่างต่อเนื่อง หากบริการภายนอกดังกล่าวไม่สามารถใช้งานได้ อาจส่งผลต่อการทำงานบางส่วนของระบบ
   6. Apportioning of requirements
ความต้องการของระบบ LHKEM ในเอกสารฉบับนี้มุ่งเน้นไปที่ฟังก์ชันหลักที่จำเป็นต่อการใช้งานPlatformในปัจจุบัน เช่น การจัดการข้อมูลชุมชน ร้านค้า Workshop การลงทะเบียนกิจกรรม และการควบคุมสิทธิ์ของผู้ใช้งานในแต่ละระดับ


ความต้องการบางส่วนที่เกี่ยวข้องกับการพัฒนาขั้นสูง เช่น ระบบชำระเงินอัตโนมัติ การตรวจสอบการชำระเงินแบบเรียลไทม์ ระบบสื่อสารภายในPlatformแบบครบวงจร หรือการวิเคราะห์ข้อมูลเชิงลึก ถูกพิจารณาเป็นความต้องการสำหรับการพัฒนาในอนาคต และไม่รวมอยู่ในขอบเขตของการพัฒนาในระยะปัจจุบัน


3. Specific requirements
   1. User Requirement Specifications
ผู้ใช้ทำอะไรกับเว็ปได้บ้าง(ฝั่งนักท่องเที่ยว)
นักท่องเที่ยวสามารถใช้งานเว็บไซต์ได้ดังนี้
1.นักท่องเที่ยวสามารถเข้าชมข้อมูลชุมชนได้โดยไม่ต้องสมัครสมาชิก
2.นักท่องเที่ยวสามารถดูรายละเอียดชุมชน ได้เเก่
* ประวัติชุมชน
* จุดเด่นทางวัฒนธรรม
* ภาพกิจกรรมและวิดีโอแนะนำ
* ข้อมูลที่ตั้งและการติดต่อ
3.นักท่องเที่ยวสามารถสมัครสมาชิกและเข้าสู่ระบบได้
4.นักท่องเที่ยวสามารถดูรายการ Workshop ทั้งหมดในชุมชน
5.นักท่องเที่ยวสามารถค้นหาและเลือก Workshop ที่สนใจได้
6.นักท่องเที่ยวสามารถดูรายละเอียด Workshop ได้เเก่
* ชื่อกิจกรรม
* วันและเวลา
* สถานที่
* จำนวนที่นั่งที่เหลืออยู่แบบ Real-time
7.นักท่องเที่ยวสามารถลงทะเบียนเข้าร่วม Workshop ได้
8.นักท่องเที่ยวสามารถตรวจสอบประวัติการลงทะเบียน Workshop ของตนเองได้
9.นักท่องเที่ยวสามารถดูแผนที่ชุมชนแบบ Interactive เพื่อค้นหาร้านค้า จุดสำคัญ และ Workshop ได้
10.นักท่องเที่ยวสามารถคลิกตำแหน่งบนแผนที่เพื่อดูรายละเอียดร้านค้าหรือกิจกรรมได้
11.นักท่องเที่ยวสามารถรับอีเมลแจ้งเตือนเกี่ยวกับ Workshop หรือกิจกรรมที่ลงทะเบียนไว้

   1. Functions
      1. Workshop Registration System
* นักท่องเที่ยวสามารถเข้าดูรายการ Workshop ต่างๆ เพื่อทำการดูรายละเอียด ข้อมูลต่างๆของ Workshop และลงทะเบียนเข้าทำ Workshop ได้
      1. Interactive Community Map
* แผนที่กราฟิกแบบสามารถโต้ตอบได้กับผู้ใช้งาน ซึ่งแสดงตำแหน่งของชุมชน ,รายละเอียด และกิจกรรม Workshop ต่างๆของชุมชนนั้นๆ
      1. User Account & Membership System
* การสมัครบัญชีผู้ใช้ สำหรับนักท่องเที่ยวเพื่อทำการเข้าใช้เงานบริการต่างๆในเว็บแอปพลิเคชั่น
* การสมัครสมาชิก ของร้านค้า/ผู้จัดกิจกรรมที่ต้องการจะเปิด Workshop ให้กับนักท่องเที่ยว
      1. Community Information Page
* แสดงข้อมูลต่างๆที่สำคัญของแต่ละชุมชน เช่น ประวัติความเป็นมา, จุดเด่น, ภาพรวม และกิจกรรมต่างๆในพื้นที่
      1. Workshop Management Module
* ร้านค้าหรือผู้จัดกิจกรรมสามารถสร้าง จัดการข้อมูลต่างๆของ Workshop ตนเองได้ เช่น รายละเอียด workshop ที่ผู้เข้าร่วมจะได้รับ รูปภาพตัวอย่าง วันและเวลาการจัดงาน สถานที่ และราคาเข้าร่วม
________________
      2. Event Management Module
* ผู้ดูแลชุมชนสามารถจัดการ Event ที่เป็นกิจกรรมรวมของชุมชนได้
      1. Shop Profile Management System
* ร้านค้าสามารถสร้างโปรไฟล์หลักเพื่อแสดงข้อมูลรายละเอียดต่างๆของร้านค้าของตน ให้ผู้ใช้งานเข้ามาสำรวจได้ เช่น รูปภาพ, ที่ตั้ง, เวลาทำการ, รายการ Workshop ของร้านค้า
      1. Email Notification System
* ระบบส่งอีเมลอัตโนมัติให้กับผู้เข้าร่วม Workshop เช่น แจ้งเตือนการยกเลิกWorkshop, การปรับเปลี่ยน รายละเอียดต่างๆของ Workshop นั้นๆที่ได้ลงทะเบียนไว้ อย่างการเปลี่ยนวันและเวลาการจัด  Workshop และสถานะการชำระเงิน พร้อมใบเสร็จ
      1. Community Admin Panel
* ผู้ดูแลชุมชนสามารถจัดการข้อมูลชุมชน ร้านค้า และกิจกรรมภายในพื้นที่ของตน เช่น การอนุมัติในการจัด Workshop และ Event ภายใต้นโยบายและข้อกำหนด
      1. Platform Admin Dashboard
* ผู้ดูแลPlatformสามารถจัดการภาพรวมทั้งเว็บไซต์ เช่น เพิ่มชุมชนใหม่, การคัดกรองสมาชิก/ผู้จัดกิจกรรม ,ดูแลความเรียบร้อยของข้อมูลต่างๆ ของผู้ใช้งาน
   1. Performance Requirements
* ระบบควรโหลดหน้าเว็บภายใน 10 วินาที
* รองรับผู้ใช้พร้อมกันได้ไม่น้อยกว่า 100 คน
* การส่งอีเมลแจ้งเตือนควรใช้เวลาไม่เกิน 5 นาทีหลังมีการเปลี่ยนแปลง

   2. Logical Database Requirements
USER: เก็บข้อมูลผู้ใช้งานทุกบทบาท (นักท่องเที่ยว / ร้านค้า / Admin)
* user_id: idผู้ใช้(Primary Key)
* firstname: ชื่อจริงผู้ใช้
* lastname: นามสกุลผู้ใช้
* password รหัสผู้ใช้
* email: อีเมลผู้ใช้
* phone: เบอร์โทร
* created_at: วันที่สมัคร
                COMMUNITY : (Long Him Kow และชุมชนอื่นในอนาคต)
* community_id: idชุมชน(Primary Key)
* name: ชื่อชุมชน
* history: ประวัติ
* address: ที่ตั้ง
* contact_info: ข้อมูลติดต่อ
* created_at: วันที่สร้าง
                COMMUNITY_ADMIN : เชื่อม Admin กับ Community
* community_admin_id : idที่เชื่อมcommunityกับadmin(Primary Key)เอาไว้ระบุ admin
* user_id: id ผู้ใช้(foreign key)
* community_id: id ชุมชน(foreign key)
                
________________
SHOP : โปรไฟล์ร้านค้า
* shop_id : id ร้านค้า (Primary Key)
* user_id : id เจ้าของร้าน (foreign key)
* community_id: id ชุมชน (foreign key)
* shop_name : ชื่อร้าน
* picture : รูปร้าน
* description : คำอธิบายร้าน
* open_time : เวลาเปิดร้าน
* contact : ช่องทางติดต่อร้าน
* created_at : วันที่สร้างร้าน
                WORKSHOP : กิจกรรมเวิร์กช็อปที่ร้านสร้าง
* workshop_id : idของworkshop(Primary Key)
* shop_id : ไอดีร้าน (foreign key)
* community_id : id ชุมชน (foreign key)
* title : ชื่อworkshop
* picture : รูป workshop
* description : คำอธิบายworkshop
* location : ตำเเหน่งworkshop
* start_datetime : เวลาเริ่ม
* end_datetime : เวลาปิด
* seat_limit : จำนวนที่นั้งทั้งหมด
* deposit_amount: ค่าลงทะเบียน
* status : เปิด/ปิด
* created_at : วันที่สร้าง
                
________________
EVENTS : กิจกรรมที่ Admin สร้าง(กิจกรรมใหญ่ในชุมชน)
* event_id : idกิจกรรม(Primary Key)
* community_id : id ชุมชน (foreign key)
* created_by : id user ที่สร้าง (foreign key)
* title : หัวข้อกิจกรรม
* picture : รูปกิจกรรม
* description : ข้อมูลกิจกรรม
* location : ตำเเหน่งกิจกรรม
* event_date : เวลาจัดกิจกรรม
* created_at : วันที่สร้างกิจกรรม
                WORKSHOP_REGISTRATION : ลงทะเบียนเวิร์กช็อป
* registration_id : id ที่ลงทะเบียน(Primary Key)
* workshop_id : id เวิร์กช็อป
* user_id : id ผู้ใช้ 
* registered_at : วันเวลาที่ลงทะเบียน
* status : สถานะ (CONFIRMED,CANCELLED)


   3. Design Constraints
* สามารถใช้ได้ในหลายแพลตฟอร์ม ทั้งคอมพิวเตอร์, แท็บเล็ต, มือถือ
* รองรับภาษาไทยและต่างประเทศ ให้กับผู้เข้าใช้งาน
* มีเพียงเว็บแอปพลิเคชั่น ยังไม่พัฒนาในส่วนแอปพลิชั่นบนมือถือ
* ต้องมีการเชื่อมต่ออินเทอร์เน็ตในการเข้าใช้งาน
* ต้องทำการล็อกอินก่อน ถึงจะใช้บริการต่างๆได้ แต่หากยังไม่ได้ล็อกอิน ก็สามารถเข้าชมเว็บไชต์ และกิจกรรมต่างๆได้
